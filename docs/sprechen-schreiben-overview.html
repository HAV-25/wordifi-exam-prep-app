SPRECHEN Speaking Assessment

How It Works

  • Live session via OpenAI Realtime API (WebRTC / WebSocket fallback)
  • AI partner — female voice (coral / shimmer / alloy), randomised female German name
  • Two modes: Dialogue (partner-led) · Monologue (user presents)
  • 4-minute max; VAD silence detection ends each turn automatically
  • Transcript built from Whisper transcriptions; sent to score-sprechen on end

Scoring Formula (Backend — deterministic)

Fluency (1–5) × 35% → max 35 pts
Grammar (1–5) × 40% → max 40 pts
Vocabulary (1–5) × 25% → max 25 pts
─────────────────────────────────
Raw Total → 0–100

Integrity Guards (applied after raw total)

< 10 unique words → score capped at 20

< 20 unique words → score capped at 40

< 2 user turns → task_completion forced to minimal

task_completion partial → × 0.8 · minimal → × 0.5

Pass Threshold

≥ 60 → Passed   < 60 → Needs Improvement   (universal across A1/A2/B1)

Content Moderation

OpenAI Moderation API runs on Sie: lines before GPT scoring. Flagged → 0/100, not saved to DB. Fails open on API errors.

SCHREIBEN Writing Assessment

How It Works

  • User writes a German response in a text editor within the app
  • Task types: Form Fill · SMS · Formal Email · Informal Letter · Opinion
  • One-attempt rule — first submission is locked and cached for the question
  • English detection guard: submission rejected before calling GPT
  • Submitted to assess-schreiben; form_fill scored locally (no GPT)

Scoring Rubric (CEFR-level specific)

A1 — Max 10 pts · Pass ≥ 6
  Task Completion (3 content points × 3.0) + Register (1.0)

A2 — Max 12 pts · Pass ≥ 7
  Sprachfunktion (5) + Register (2) + Spektrum (2.5) + Beherrschung (2.5)

B1 — Max 15 pts · Pass ≥ 9
  Erfüllung (4) + Textsorte (1) + Register (1) + Kohärenz (1.5)
  + Wortschatz Spektrum (2) + Wortschatz Beherr. (1.5)
  + Strukturen Spektrum (2) + Strukturen Beherr. (2)

Integrity Guards

English submission → 0 pts, not saved, no GPT call

> 20% English in German text → language mixing penalty applied by GPT

Moderation API runs before GPT — flagged → 0 pts, not saved

Pass Threshold

A1 ≥ 6/10   A2 ≥ 7/12   B1 ≥ 9/15

Content Moderation

OpenAI Moderation API runs on user_text before any DB read or GPT call. Applies to all task types including form_fill. Fails open on API errors.

Shared Infrastructure

Moderation (both)

  • Model: omni-moderation-latest
  • Checks: harassment, hate, sexual, violence, self-harm, illicit
  • Fail-open — transient API errors never block the user
  • Logs: categories fired + top scores (no user content logged)
  • Blocked responses never saved to DB

Diagnostic Logging (Sprechen)

  • total_words · unique_words
  • user_turns · raw_contribution_total
  • final_multiplier (1.0 / 0.8 / 0.5)
  • capped_by_guard boolean
  • overall_score · passed · task_completion

Edge Functions

  • assess-schreiben v10 — scoring + moderation
  • DB: sprechen_responses · schreiben_submissions
  • Analytics via track('section_completed') with score_pct
